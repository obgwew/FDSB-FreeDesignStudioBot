# هيكل المشروع - File & Bot Manager

## بنية الملفات

```
project_root/
├── main.py                      # الملف الرئيسي
├── config.py                   # إعدادات التطبيق والألوان
├── translations.py             # إدارة الترجمات
├── data_manager.py             # إدارة البيانات والملفات
├── widgets.py                  # الأدوات المخصصة والنوافذ المنبثقة
├── main_screen.py             # الشاشة الرئيسية
├── file_editor_screen.py      # شاشة محرر الملفات
├── files_list_screen.py       # شاشة قائمة الملفات
├── variables_screen.py        # شاشة إدارة المتغيرات
├── local_server.py           # سيرفر البوت المحلي
├── requirements.txt          # متطلبات المشروع
├── buildozer.spec           # إعدادات buildozer للأندرويد
├── app_data/                # مجلد البيانات
│   ├── files/              # ملفات المستخدم
│   ├── config.json         # إعدادات التطبيق
│   ├── variables.json      # المتغيرات
│   └── prefixes.json       # بادئات الملفات
└── bot_token.txt           # توكن البوت
```

## ميزات التقسيم

### 1. سهولة الصيانة
- كل شاشة في ملف منفصل
- فصل المنطق عن واجهة المستخدم
- إدارة مركزية للألوان والإعدادات

### 2. قابلية التوسع
- إضافة شاشات جديدة بسهولة
- تعديل التصميم من ملف واحد
- إضافة لغات جديدة بسهولة

### 3. إعادة الاستخدام
- أدوات مخصصة قابلة للاستخدام
- نظام ألوان موحد
- نظام ترجمة شامل

## نصائح التطوير

### إضافة شاشة جديدة:
1. أنشئ ملف جديد مثل `new_screen.py`
2. اجعلها ترث من `Screen`
3. استخدم `AppConfig` للألوان والأحجام
4. استخدم `Translations` للنصوص
5. أضفها في `main.py`

### تعديل الألوان:
- عدل في `config.py` فقط
- سيتم تطبيق التغيير على كامل التطبيق

### إضافة ترجمات:
- أضف النصوص في `translations.py`
- استخدم `Translations.get()` في الشاشات

### إضافة أداة جديدة:
- أضف في `widgets.py`
- اجعلها ترث من أداة Kivy الأساسية
- استخدم `AppConfig` للتصميم

1. **استخدم Clock.schedule_once** للعمليات المؤجلة
2. **احرص على تحرير الذاكرة** عند الانتقال بين الشاشات
3. **استخدم ScrollView** للقوائم الطويلة
4. **فصل العمليات الثقيلة** في threads منفصلة
